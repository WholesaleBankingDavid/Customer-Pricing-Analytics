# Dashboard Concept

The MVP dashboard should support repeated RM and pricing team workflows.

## Deal Pipeline View

- Ranked active deals by priority score.
- Win probability, total expected profit, and expected deal value.
- Pricing zone and guardrail flags.
- Filters for RM, region, product type, rating, and expected close date.

## Deal Detail View

- Facility-level economics.
- Margin above floor and expected profit contribution.
- Key model drivers.
- RM feedback and recommendation acceptance status.

## Governance View

- Deals below floor.
- Deals near guardrail.
- Exceptions requiring pricing justification.
- Outcome tracking for accepted and rejected guidance.

The dashboard should avoid presenting guidance as automated pricing. It should make assumptions, guardrails, and user overrides visible.
